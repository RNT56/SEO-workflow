# Report Contract

Every scan must produce one complete audit run directory, the required report files and the 27 mandatory sections from the SEO Polish Report contract.

The audit run directory must include:

- `index.md`
- `index.html`
- `findings.json`
- `score.json`
- `remediation-plan.json`
- `validation.json`
- `evidence.jsonl`
- `report-dashboard.json`
- `agent-review-input.json`
- `agent-review.json`
- `quality-gate.json`
- `workflow-retrospective-input.json`
- `workflow-retrospective.json`
- `workflow-retrospective.md`
- `workflow-completion.json`
- `workflow-learnings/`
- `audit-run.json`
- `priority-action-plan.md`
- `agent-execution-plan.md`
- `standards-registry.json`
- `agent-instructions/`

Portable packages generated by `seo-polish export` must include:

- `export-manifest.json`
- `checksums.sha256`
- `LICENSE-NOTICE.md`

Every finding must include:

- ID
- title
- severity
- category
- confidence
- affected URLs or templates
- safe-to-auto-fix flag
- approval-required flag
- actionability owner
- automation readiness
- source candidates or explicit source blocker
- problem
- evidence
- impact
- root cause
- recommended fix
- implementation path
- validation steps

No evidence means no finding.
